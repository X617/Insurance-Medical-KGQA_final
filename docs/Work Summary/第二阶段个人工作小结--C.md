### 二、 遇到的问题及解决思路

**1. 跨域实体链指中的歧义与冷启动问题**

- **问题描述**：医疗实体（如“糖尿病”）在保险条款中可能对应不同的给付等级，而用户输入往往模糊。单纯的向量检索在处理这类专业壁垒极高的实体时，容易出现链指错误，导致后续检索偏差。
- **解决思路**：我引入了“候选实体打分机制”。在检索图谱前，先提取候选实体集合，利用 LLM 的推理能力结合上下文进行二轮判别，确保链指到图谱节点的准确性。这一方案显著提升了系统在处理复杂保险条款时的回答精确度。

**2. 稠密图谱检索下的上下文噪声干扰**

- **问题描述**：在某些热门疾病节点（如“高血压”）上，关联的关系边极多。如果将所有关联的三元组全部塞入 Prompt，会导致上下文冗余，干扰大模型的判断，甚至触发模型的幻觉。
- **解决思路**：我设计了一套基于“意图相关性”的剪枝算法。通过计算召回路径与用户 Query 的语义重合度，仅保留权重最高的前 N 条路径。这一举措不仅降低了 API 的首字延迟，也大幅提升了生成结果的逻辑严密性。
